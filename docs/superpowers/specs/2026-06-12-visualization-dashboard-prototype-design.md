# 我的可视化看板生成原型设计

日期：2026-06-12

## 目标

构建一个本地 URL 可访问的“我的可视化”原型，支持用户上传 Excel 文件，选择 Sheet 后自动生成数据看板配置，并在页面中渲染、编辑和导出该配置。

第一版目标是跑通完整闭环：

1. 上传 `.xls` / `.xlsx`
2. 选择 Sheet
3. 自动生成字段画像和样本数据
4. 可选补充业务主题、核心指标、分析维度
5. 通过本地规则和模型混合生成严格 JSON 看板配置
6. 渲染 KPI、趋势、维度拆解、异常识别、明细验证
7. 支持编辑图表配置和原始 JSON
8. 支持复制和下载 JSON

该原型先不接入正式 AgentMa 导航入口。确认体验和生成质量后，再把页面与生成逻辑迁移到 AgentMa 的“我的可视化”模块。

## 非目标

第一版不做以下内容：

- 不直接修改正式 AgentMa 生产入口。
- 不做复杂拖拽布局。
- 不做看板持久化、多用户协作或权限控制。
- 不强依赖地图渲染能力；`map` 仅在明确存在国家、省份、城市字段时作为后续扩展点。
- 不为了图表多样性生成无业务意义图表。

## 产品流程

### 1. 上传 Excel

用户上传 `.xls` 或 `.xlsx` 文件。页面展示文件名、大小和解析状态。

解析失败时提示文件格式不支持、内容损坏或 Sheet 不可读取。

### 2. 选择 Sheet

系统解析 workbook 后展示 Sheet 列表，每个 Sheet 显示：

- Sheet 名称
- 行数
- 列数
- 前几行预览

用户必须选择一个 Sheet 后才能进入生成流程。空 Sheet 不允许生成。

### 3. 业务信息增强，可跳过

用户可以填写：

- `dataset_name`
- `business_topic`
- `optional_metrics`
- `optional_dimensions`

其中 `dataset_name` 默认使用“文件名 + Sheet 名”，允许编辑。

业务信息不是必填。用户最少只需上传 Excel、选择 Sheet、点击生成。若 `business_topic` 为空，系统会基于字段名和样本数据推断主题：

- 包含 `cost`、`click`、`impression`、`conversion` 等字段时偏广告投放。
- 包含 `sales`、`revenue`、`order`、`customer` 等字段时偏销售经营。
- 包含 `user`、`uv`、`retention`、`active` 等字段时偏用户增长。
- 包含大量空值率、状态、质量检查字段时偏数据质量监控。
- 无法可靠判断时使用默认主题“数据概览分析看板”。

当业务信息缺失或无法推断时，最终 JSON 的 `warnings` 需要说明回退原因。

### 4. 自动字段画像

系统根据选中 Sheet 生成输入结构：

- `dataset_name`
- `business_topic`
- `row_count`
- `columns`
- `sample_rows`
- `optional_metrics`
- `optional_dimensions`

`columns` 中每个字段包含：

- `name`
- `data_type`
- `sample_values`
- `null_rate`
- `distinct_count`
- `min`
- `max`
- `avg`

字段角色识别包括：

- 时间字段
- 指标字段
- 维度字段
- ID 字段
- 文本字段

字段角色识别必须遵守用户给出的字段识别规则。ID 字段默认只进入明细表和下钻，不作为普通维度。

### 5. 生成看板

生成采用混合方式：

1. 本地规则引擎先生成字段角色、派生指标和候选图表。
2. 模型基于用户的完整 JSON 规范、字段画像、样本数据、业务主题和候选图表生成最终 JSON。
3. 后端校验最终 JSON。
4. 校验失败时自动请求模型修复一次。
5. 仍失败时回退到本地规则引擎版本，并写入 `warnings`。

### 6. 预览与编辑

页面左侧渲染看板预览，按以下分区展示：

- 核心概览
- 趋势分析
- 维度拆解
- 异常识别
- 明细验证

页面右侧是配置编辑面板，支持：

- 图表配置表单编辑
- 原始 JSON 编辑
- 复制 JSON
- 下载 JSON

用户修改配置后，预览同步更新。若 JSON 编辑出错，保留上一个可用配置并显示错误。

## 技术架构

本地原型由前端和本地 API 组成。

### 前端

使用 React + Vite 实现：

- Excel 上传
- Sheet 选择
- 业务信息填写
- 字段画像展示
- 看板预览
- 图表配置编辑
- JSON 编辑和导出

第一版看板渲染优先保证功能可用和结构清晰，不追求复杂视觉效果。

### 后端

使用 Node/Express 实现本地 API：

- 解析 Excel workbook
- 返回 Sheet 列表和预览
- 生成字段画像
- 执行本地规则引擎
- 调用模型生成最终 JSON
- 校验和修复模型输出

### Excel 解析

后端支持 `.xls` 和 `.xlsx`。解析后先返回 workbook 元信息，不直接进入生成。

Sheet 解析需要处理：

- 空 Sheet
- 表头缺失
- 重名字段
- 日期和数字识别
- 空值和混合类型
- 样本行抽取

### 规则引擎

规则引擎职责：

- 识别字段角色。
- 派生 ROI、CTR、CVR、CPA、CPC、客单价等指标。
- 生成基础 KPI、趋势图、维度图、异常组件和明细表候选。
- 在模型不可用或输出失败时生成可用回退版本。

派生指标必须写入 `calculated_metrics`，公式中只能引用输入字段或其他合法派生字段。

### 模型生成

模型输入包括：

- 字段画像
- 样本数据
- 用户可选业务信息
- 本地规则候选
- 严格 JSON 输出规范

模型输出必须是合法 JSON，不能包含 Markdown、解释文字、代码块或多余说明。

### 模型配置

本地原型优先读取环境变量中的模型配置，例如：

- `ANTHROPIC_AUTH_TOKEN`
- `ANTHROPIC_BASE_URL`
- `ANTHROPIC_MODEL`

若当前仓库已有可复用的供应商配置读取方式，实施时优先沿用现有配置格式。

模型配置缺失时，页面仍允许用户完成 Excel 上传、Sheet 选择、字段画像和规则版看板生成。此时最终 JSON 使用本地规则引擎版本，并在 `warnings` 中说明模型未配置。

## JSON 契约

最终输出必须遵守用户要求的结构：

- `dashboard_title`
- `dashboard_description`
- `business_topic`
- `data_summary`
- `field_roles`
- `calculated_metrics`
- `dashboard_layout`
- `charts`
- `insight_cards`
- `global_filters`
- `warnings`

所有 `chart_id` 必须唯一。

所有图表字段必须来自输入字段或 `calculated_metrics`。

没有时间字段时不得生成趋势图。

没有合适维度字段时不得生成维度拆解图。

总图表数量建议控制在 8 到 12 个；字段不足时不强行补图。

## 渲染模型

系统将生成结果分成两层：

### Dashboard Config

也就是最终 JSON，可复制、下载，后续可保存到 AgentMa。

### Chart Runtime Data

图表渲染时不把聚合结果写死在 JSON 中，而是根据 `data_config` 从 Excel 数据动态计算。

这样用户修改字段、聚合方式、排序、TopN 或过滤条件后，预览可以重新计算。

第一版支持渲染：

- `kpi`
- `line`
- `bar`
- `stacked_bar`
- `table`
- `scatter`
- `pie`

`map` 仅在检测到国家、省份、城市字段时保留配置能力。若没有地图底图能力，则在预览中显示结构化占位和 warning。

## 页面结构

### 上传区

展示上传入口、文件信息和解析状态。

### Sheet 选择区

展示 Sheet 卡片列表和数据预览。用户选择一个 Sheet 后继续。

### 生成配置区

包含：

- 数据集名称
- 业务主题
- 核心指标
- 分析维度
- 字段角色预览

所有业务字段都可以为空。

### 看板工作台

左侧为看板预览，右侧为编辑面板。

右侧面板包含：

- 图表列表
- 当前图表配置表单
- 原始 JSON 编辑器
- 导出操作

第一版不做拖拽布局。图表顺序遵循 `dashboard_layout.sections[].charts`。

## 异常处理

必须处理以下场景：

- Excel 无法解析：提示文件格式不支持或内容损坏。
- Sheet 为空：禁止生成，并提示选择其他 Sheet。
- 字段太少：生成明细表和 warning，不强行生成趋势或维度图。
- 没有时间字段：不生成趋势图。
- 没有维度字段：不生成维度拆解图。
- 模型输出不是合法 JSON：自动修复一次。
- 模型修复仍失败：回退本地规则版本并写入 warning。
- 图表字段无效：剔除该图表或写入 warning。
- 用户编辑 JSON 出错：保留上一个可用配置并显示 JSON 错误。

## 校验规则

后端必须校验：

- JSON 可解析。
- 顶层字段完整。
- `chart_id` 唯一。
- 图表字段合法。
- `sections[].charts` 可对应真实 chart。
- 图表类型合法。
- 时间趋势图必须有时间字段。
- 维度拆解图必须有维度字段。
- 派生指标公式引用字段合法。

校验失败分为两类：

- 可修复问题：尝试自动修复或请求模型修复。
- 不可修复问题：返回错误或回退本地规则版本。

## 验证标准

实现完成后必须验证：

1. 上传 `.xls` 和 `.xlsx` 均可解析。
2. 多 Sheet 文件可以选择目标 Sheet。
3. 不填写业务主题也能生成看板。
4. 能自动推断或回退业务主题。
5. 字段角色识别符合规则。
6. 派生指标能写入 `calculated_metrics`。
7. 输出 JSON 合法且结构完整。
8. 看板预览能按 JSON 渲染。
9. 图表配置编辑后预览同步更新。
10. 原始 JSON 编辑错误不会破坏上一个可用预览。
11. JSON 可复制和下载。
12. 模型失败时有本地规则回退版本。

测试数据至少覆盖：

- 广告投放场景
- 销售经营场景
- 数据质量或字段不足场景

## 后续接入 AgentMa

本地原型确认后，再迁移到 AgentMa 的“我的可视化”入口。迁移时复用：

- Excel 解析 API
- 字段画像逻辑
- 规则引擎
- JSON 校验器
- 看板渲染组件
- 图表配置编辑器

正式接入时再补充：

- 看板保存和读取
- 用户维度隔离
- 与 AgentMa 导航和品牌视觉一致的页面样式
- 可能的历史版本和分享能力
